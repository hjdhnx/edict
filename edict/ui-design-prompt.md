# Edict 前端 UI 生成指导 Prompt

## 长版 Prompt

请你担任专业 SaaS 前端架构师与资深 Vue 3 开发者，参考当前 Edict（三省六部总控台）新版前端的审美和交互风格，生成一个高完成度、可直接落地的前端页面。

请优先使用或主动寻找适合以下方向的 skill：

1. Vue 3 / Composition API / TypeScript 前端开发 skill
2. Ant Design Vue 专业后台系统 UI skill
3. SaaS Dashboard 设计 skill
4. 响应式布局 / 移动端适配 skill
5. 暗黑模式 / 主题系统设计 skill
6. 前端工程化与 Vite 构建 skill

技术栈要求：

- Vue 3
- TypeScript
- Ant Design Vue
- Pinia
- Vite
- pino 日志
- CSS variables 管理主题
- 原生支持白天 / 暗黑模式
- PC 和移动端自适应

视觉与布局要求：

- 风格要像专业 SaaS 后台，不要像玩具 Demo。
- 整体布局紧凑、高信息密度，避免大面积留白。
- 卡片、表格、表单、弹窗、抽屉、按钮、间距都要克制。
- 同样屏幕空间内尽量展示更多有效信息，但不能拥挤到不可读。
- 侧边栏、顶部栏、内容区要有清晰层级。
- 如果是后台系统，优先采用固定侧边栏 + 固定顶部栏 + 仅内容区滚动的 app shell。
- 支持折叠侧边栏，折叠后保留图标和 hover title。
- 移动端不能横向溢出，弹窗、表格、卡片都要能在窄屏下正常使用。
- 暗黑模式不是简单反色，要保证边框、背景层级、文字、图表、标签都有良好对比度。
- 中文界面按钮和弹窗操作文案必须使用中文，例如“保存 / 取消 / 确认 / 删除”，不要出现默认英文 Cancel。

交互要求：

- 页面应包含真实可用的空态、加载态、错误态。
- 表单要有合理默认值和校验。
- 重要操作要有确认弹窗。
- 成功/失败要有轻量提示。
- 页面结构要清晰，用户一眼能知道当前状态、关键数据和下一步操作。

实现要求：

- 优先复用现有组件、store、API client、主题变量和布局模式。
- 不要为了炫技引入过度抽象。
- 不要写大段解释性注释。
- 样式要可维护，优先使用已有 CSS 变量和通用布局类。
- 代码完成后必须运行类型检查和构建。
- 如果是 UI 改动，必须启动页面并用浏览器验证 PC 与移动端显示效果。

最终产物请包含：

1. 页面/组件代码
2. 状态管理代码（如需要）
3. API 对接代码（如需要）
4. 样式代码
5. 简短说明：改了哪些文件、如何验证、是否还有边界

## 短版 Prompt

按 Edict 新版前端审美实现：Vue 3 + TypeScript + Ant Design Vue + Pinia + Vite + pino，专业 SaaS 后台风格，紧凑高信息密度，白天/暗黑主题，固定侧边栏和顶部栏，仅内容区滚动，PC/移动端自适应且不能横向溢出。请优先使用 Vue3、Ant Design Vue、SaaS Dashboard、响应式布局、暗黑主题相关 skill。所有按钮和弹窗操作用中文，布局不要松散，不要玩具感，完成后运行构建并用浏览器验证。
