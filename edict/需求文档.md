# Edict 生产力化改造需求文档

> 来源：`项目价值报告.md`。本文档是本次长任务的活文档；每完成一个任务，都必须立即更新对应状态与完成记录。

## 1. 目标

把 Edict 从“核心链路有价值、外围功能部分占位”的状态，推进为更诚实、更可靠、更适合本地长期使用的 AI Agent 工单系统与审计看板。

## 2. 范围

### 2.1 本轮优先范围

1. 显式处理 stub 功能，避免用户误以为已生效。
2. 强化 Agent 完成产物规范，让完成态必须携带可审计产物或说明。
3. 接入或准备接入 WebSocket，降低纯轮询依赖。
4. 清理安全配置，避免提交真实密钥与开发默认值误导。
5. 校准数据库迁移与 ORM 模型，保证新环境可初始化关键 worker 依赖。
6. 清理明显陈旧的文档或 UI 文案。

### 2.2 非目标

- 不把系统改造成公网 SaaS。
- 不引入完整多用户登录权限体系。
- 不重写三省六部业务模型。
- 不新增未在价值报告中体现的新大功能。

## 3. 验收标准

- 用户能区分“已生效配置”和“暂未启用能力”。
- Agent 完成任务时，最终产物在任务详情/奏折中可追踪。
- 前端可使用 WebSocket 接收实时事件，失败时仍能退回轮询。
- Compose 与示例环境不再包含真实密钥。
- Alembic 初始迁移包含当前 worker 主链路依赖的表结构。
- 文档和 UI 不再宣传当前代码未实现的能力。
