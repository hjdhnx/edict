# Edict 前端重构需求文档

> 本文档用于跟踪本轮前端整体替换长任务。用户已确认迁移策略为“整体替换”，视觉方向为“专业 SaaS”。每完成一个阶段，必须更新状态与完成记录。

## 1. 目标

将现有 React 自定义样式前端整体替换为更现代、专业、可长期维护的 Vue 3 SaaS 控制台，同时保持现有后端 API、WebSocket、Docker/nginx 部署契约不变。

## 2. 技术栈

- Vue 3 + Composition API + `<script setup lang="ts">`
- Vite
- TypeScript
- Ant Design Vue
- Tailwind CSS
- Pinia
- pino 浏览器日志

## 3. 范围

### 3.1 必做

1. 整体替换 React 前端入口、组件与状态管理。
2. 保留现有 API 类型与调用契约。
3. 保留 `/ws` 实时事件订阅，并保留 5 秒轮询兜底。
4. 实现专业 SaaS 风格布局：侧边导航、顶部状态栏、卡片化信息层级、清晰操作区。
5. 原生支持白天/暗黑模式切换，并持久化用户选择。
6. 支持 PC 与移动端自适应。
7. 保留现有核心业务模块：旨意看板、任务详情、奏折阁、天下要闻、朝堂议政、官员/模型/技能/监控等配置与看板能力。
8. 接入 pino 日志，覆盖 API、WebSocket、关键用户操作与异常。
9. 更新 Docker 构建流程，保持 nginx 静态托管、`/api` 与 `/ws` 代理可用。
10. 安装并使用前端开发与 Web 应用测试相关 skill。

### 3.2 非目标

- 不改造后端业务接口。
- 不新增登录/多租户体系。
- 不把当前三省六部业务模型改成通用项目管理系统。
- 不引入额外 UI 框架与 Ant Design Vue 竞争。

## 4. 验收标准

- `npm run build` 通过。
- Docker 前端镜像可重建，`http://127.0.0.1:7899/` 可访问。
- 前端 nginx 代理 `/api/live-status` 正常返回。
- `/ws` WebSocket 可握手，前端收到事件后能刷新数据。
- 白天/暗黑主题可切换并刷新后保持。
- PC 宽屏、平板、手机宽度下核心页面可读、可操作。
- 下旨、推进、归档、任务详情、奏折查看、天下要闻刷新、模型/技能配置等核心路径可用。
- 控制台无明显运行时报错。
